Product Requirements Document: Helios

Author: Priya Nair (Product Manager)
Date: 2026-02-03
Status: Approved

1. Overview

Helios is our company's payments platform. Helios lets customers sign in and pay
merchants using cards they have saved. This document defines the product requirements
for the two launch features of Helios: User Authentication and Payment Processing.

2. Features

Helios ships with two features at launch.

Feature F1 - User Authentication is part of Helios. User Authentication covers how a
customer proves who they are before they can transact.

Feature F2 - Payment Processing is part of Helios. Payment Processing covers how a
customer pays a merchant once they are authenticated.

3. Functional Requirements

REQ-1: Users authenticate with email and password. This requirement specifies the
User Authentication feature. Priority: must-have. This is a functional requirement.

REQ-2: Users complete a payment with a saved card. This requirement specifies the
Payment Processing feature. Priority: must-have. This is a functional requirement.

REQ-3: Users reset their password via an emailed link. This requirement specifies the
User Authentication feature. Priority: should-have. This is a functional requirement.

4. Non-Functional Requirements

REQ-4: The login endpoint sustains 10,000 requests/second at p99 < 200 ms. This
non-functional requirement specifies the User Authentication feature. Metric:
login throughput and p99 latency. Target value: 10,000 requests/second at p99 < 200 ms.
Priority: must-have. This is a performance NFR.

5. Notes on Implementation Direction

At a high level, sign-in traffic is served by the auth-service, and payment traffic is
served by the payment-service. Detailed design lives in the HLD and the per-service LLDs.
